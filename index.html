Structural elements

I'm not 100% sure what's going on with all of these, but here they are.

Wrappers

Tabzilla demands that:

  1. Background images must not be attached to the element.
  2. Absolutely positioned elements must not be positioned relative to the element.
  3. An element other than the should add a 2 pixel white border to the top of the page (border-top: 2px solid #fff;)

We use two wrapper divs—#outer-wrapper and #wrapper—to satisfy the requirements. Except currently the #outer-wrapper code is broken, so there's no top border. Oops!

Attach background images to #wrapper. Notice that the background moves down as tabzilla opens.

Containers and rows

.container divs are a non-responsive fixed width. .row divs provide a negative margin to properly align their .span* divs. (You can plonk content straight in a .container, but not a .row without alignment issues.)

Site navigation

Site nav is not particularly high tech. Each page sets its own id on the body tag, and CSS selectors take over from there. We don't expect a lot of new pages, or something fancier and javascriptier could have been done.

Gallery viewing

Thimble has will have two galleries—Projects and Gallery—for learning projects and user creations respectively. They each look a little different. The markup for both is Bootstrap-inspired, though Bootstrap probably does extra cool things I don't know about and didn't steal.

Projects page

The Projects gallery has both list and thumbnail views, togglable by setting .show-list or .show-thumbs on the containing <ul class="projects">. projects.js will do that for you in a very basic sort of way, if you have jQuery already included on the page.
